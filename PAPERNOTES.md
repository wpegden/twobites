## Corrections And Clarifications
- Global small-`ε` reduction: the proof repeatedly assumes `0 < ε_2 << ε_1 << ε << 1` and later uses inequalities that require `ε` to be sufficiently small, for example `n^{4ε} k = o(k^2)`, `1 - 8ε > 0`, and `-1 + ε + 22ε^2 < 0`. To match Theorem `main` as stated for all `ε > 0`, the paper should explicitly say that it suffices to prove the result for sufficiently small `ε`, since the conclusion is monotone in `ε`.
- `reference.tex:331-334` (`lem:fiber_and_degree`): the proof is only sketched. For Lean planning, this should be split into routine concentration components: fiber size concentration, degree concentration in `G_R`/`G_B`, concentration of `|N_v|`, codegree bounds `|N_v ∩ N_w| = O(log^3 n)`, and projected codegree bounds. The projected codegree estimates use sampling without replacement in the blue/red coordinates.
- `reference.tex:421-422` (`lem:med`): the second bipartite graph should be on `(B_B, π_B(I))`, not `(B_B, π_R(I))`.
- `reference.tex:436` (`lem:med`): the final probability bound has the wrong event. The argument gives a bound by `P(\mathcal O \cup \mathcal D^c)`, not `P(\mathcal O^c \cup \mathcal D^c)`.
- `reference.tex:477-491` (`lem:small`): the McDiarmid step is valid, but only after making the independence mechanism explicit. For `v ∉ π_R(I) ∪ π_B(I)`, the variable `Y_v` depends only on edges between `v` and `π_R(I)` or `π_B(I)`; these edge sets are disjoint across different `v`, so the `Y_v` are independent.
- `reference.tex:557` (`lem:huge`): `N_3(v_1)` is undefined. The intended bound is `x_{v_1} ≤ |N_{v_1}| ≤ (1 + ε_2)pn`, or equivalently `|\pi_B(X_{v_1}(I))| ≤ (1 + ε_2)pn`.
- `reference.tex:578-580` (`lem:huge`): the expression `π(I) \setminus (\bigcup X_{v_i}(I))` is ill-typed because `π(I) ⊆ V_R × V_B` while `X_{v_i}(I) ⊆ V(G)`. The intended inequality is
  `|π_R(I)| ≤ |π_R(\bigcup X_{v_i}(I))| + |I \setminus \bigcup X_{v_i}(I)|`,
  which then yields `|\bigcup X_{v_i}(I)| ≤ k - |π_R(I)| + o(k)`.
- `reference.tex:533-580` (`lem:huge`): beyond the two corrections above, the manuscript keeps the huge-case union-size and projected-codegree losses only at qualitative `o(k)` scale. It never names explicit blue/red slack parameters for those two witness-error pieces. The Lean development therefore has to expose exact branchwise smallness hypotheses for those pieces instead of recovering a single canonical constant from the text. With only the displayed scale assumptions, one can formalize at best the coarse bounds `paperHugeWitnessDegreeBranchParam ≤ 3κ`, `paperHugeWitnessCodegBranchParam ≤ 3κ`, and hence `paperHugeWitnessBranchParam ≤ 6κ`; the genuinely small branch slacks needed by the final huge-case route require an extra large-`n` specialization that the manuscript leaves implicit. Both qualitative pieces are now quantified separately in Lean via `paperHugeWitnessDegreeBranchParam_eventually_le` and `paperHugeWitnessCodegBranchParam_eventually_le`, and the combination lemmas `paperHugeWitnessBranchwisePieceBranchParamBounds_eventually_le` / `paperHugeWitnessBranchwiseBranchParamBounds_eventually_le` package them into common-threshold blue/red branch-slack bounds. The current concrete Lean specialization takes each exact piece to be at most `ε₁ κ`, hence each branch slack to be at most `2 ε₁ κ`; this reconnects the sharpened exact huge-case route under the stronger budget `ρ + (1+ε₂)β + 4 ε₁ κ + δsumGap + δgap ≤ κ`. Lean now packages that specialization behind the canonical threshold `paperHugeWitnessTwoEpsBranchPieceThreshold ε₁ κ βdeg qcodeg`, so the exact huge-case theorem can be applied with one explicit `n`-largeness hypothesis instead of a separately chosen witness `N`. More precisely, the comparison wrappers `...of_eventualTwoEpsBranchPieceBounds_of_extraDeficit`, `...of_twoEpsBranchPieceThreshold_of_paramDeficit`, `...of_twoEpsBranchPieceThreshold_of_fourEpsBudget`, `...of_twoEpsBranchPieceThreshold_of_rhoBudget`, `...of_twoEpsBranchPieceThreshold_of_coarseBudget`, and the higher compatibility wrappers `...of_twoEpsBranchPieceThreshold_of_witnessCoeff_of_paramDeficit` / `...of_twoEpsBranchPieceThreshold_of_exactWitnessCoeff_of_paramDeficit` together with `...of_twoEpsBranchPieceThreshold_of_witnessCoeff_of_fourEpsBudget` / `...of_twoEpsBranchPieceThreshold_of_exactWitnessCoeff_of_fourEpsBudget` and `...of_twoEpsBranchPieceThreshold_of_witnessCoeff_of_rhoBudget` / `...of_twoEpsBranchPieceThreshold_of_exactWitnessCoeff_of_rhoBudget` show that the sharpened route can replace the older coarse `6 * paperK κ` path as soon as the later paper arithmetic provides either branchwise deficit parameters `δblue, δred` satisfying `2 ε₁ κ ≤ δblue, δred` and `ρ_R + (1+ε₂)β + 2 ε₁ κ + δblue + δsumGap + δgap_R ≤ κ`, `ρ_B + (1+ε₂)β + 2 ε₁ κ + δred + δsumGap + δgap_B ≤ κ`, or equivalently the direct strengthened branch budgets `ρ_R + (1+ε₂)β + 4 ε₁ κ + δsumGap + δgap_R ≤ κ`, `ρ_B + (1+ε₂)β + 4 ε₁ κ + δsumGap + δgap_B ≤ κ`, or equivalently the direct bounds `ρ_R ≤ paperHugeWitnessFourEpsRhoBudget β ε₁ ε₂ κ δsumGap δgap_R`, `ρ_B ≤ paperHugeWitnessFourEpsRhoBudget β ε₁ ε₂ κ δsumGap δgap_B`. Lean now also has the explicit manuscript-style specialization `β = 1/2`, `κ = 1 + ε`: if `ρ ≤ ((1/2) + ε/4)(1 + ε)` and the lower-order losses satisfy `ε₂ / 2 + 4 ε₁ (1 + ε) + δsumGap + δgap ≤ ε (1 - ε) / 4`, then automatically `ρ ≤ paperHugeWitnessFourEpsRhoBudget (1/2) ε₁ ε₂ (1 + ε) δsumGap δgap`. This makes the remaining issue sharper: in the middle regime of `lem:RI`, the manuscript only visibly forces the smaller projection parameter (after relabeling, `x_B`) below `1/2 + ε/4`, so it is still unclear whether the later argument really supplies the symmetric two-branch quarter-projection control needed to replace the coarse huge-case route outright, or whether one should instead formalize an asymmetric sharpened route matching the proof as written.
- `reference.tex:721-731` (middle case of `lem:RI`): Lean now has the asymmetric sharpened huge-case route in the orientation dictated by the manuscript's relabeling step. The arithmetic lemmas `paperHugeWitnessQuarterProjection_of_middleSum_of_blueLeRed`, `paperHugeWitnessQuarterX_of_middleSum_of_blueLeRed`, and `paperHugeWitnessQuarterProjection_of_xMiddleSum_of_blueLeRed` formalize the implications
  `ρ_B ≤ ρ_R` and `ρ_R + ρ_B ≤ (1 + ε/2)(1 + ε)` `⇒` `ρ_B ≤ ((1/2) + ε/4)(1 + ε)`,
  together with the direct `x_B ≤ x_R`, `x_R + x_B ≤ 1 + ε/2` `⇒` `x_B ≤ 1/2 + ε/4` step used in the manuscript. `Twobites/IndependentSets.lean` packages the resulting one-sided huge-case theorems as `...of_halfBeta_of_onePlusEps_of_middleRange_of_blueLeRed` and `...of_halfBeta_of_onePlusEps_of_xMiddleRange_of_blueLeRed`, together with witness-/exact-witness compatibility versions. This resolves the orientation question in favor of the relabeled smaller `B`-branch. The exact remaining coarse dependency is now explicit: these paper-aligned middle-range wrappers still require the opposite-branch `kSmall` hypothesis (`hblueCrossSmall`), and there are no higher direct uses of the old symmetric `...of_kSmall...` theorem left above this interface.
- `reference.tex:721-731` (continuation of the middle-case issue): the surviving opposite-branch coarse hypothesis is not just currently unproved from the later manuscript arithmetic; it is formally incompatible with the present cross-residual shape. `Twobites/ParameterBounds.lean` proves `cross_residual_sub_one_le_paperK`, hence `not_six_mul_paperK_le_cross_residual`, and in the paper's specialized regime `β = 1/2`, `κ = 1 + ε` this becomes `not_six_mul_paperK_le_cross_residual_half_one_add_eps`. Therefore the `hblueCrossSmall` input still present in the new middle-range wrappers cannot be discharged by any later arithmetic of the same residual form. The remaining formalization task is to replace that opposite-branch `6 * paperK κ` condition by a genuinely smaller witness-error statement, or to redesign the middle-case huge theorem so it no longer asks for an impossible coarse residual.
