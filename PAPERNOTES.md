## Corrections And Clarifications
- Global small-`Оө` reduction: the proof repeatedly assumes `0 < Оө_2 << Оө_1 << Оө << 1` and later uses inequalities that require `Оө` to be sufficiently small, for example `n^{4Оө} k = o(k^2)`, `1 - 8Оө > 0`, and `-1 + Оө + 22Оө^2 < 0`. To match Theorem `main` as stated for all `Оө > 0`, the paper should explicitly say that it suffices to prove the result for sufficiently small `Оө`, since the conclusion is monotone in `Оө`.
- `reference.tex:331-334` (`lem:fiber_and_degree`): the proof is only sketched. For Lean planning, this should be split into routine concentration components: fiber size concentration, degree concentration in `G_R`/`G_B`, concentration of `|N_v|`, codegree bounds `|N_v вҲ© N_w| = O(log^3 n)`, and projected codegree bounds. The projected codegree estimates use sampling without replacement in the blue/red coordinates.
- `reference.tex:421-422` (`lem:med`): the second bipartite graph should be on `(B_B, ПҖ_B(I))`, not `(B_B, ПҖ_R(I))`.
- `reference.tex:436` (`lem:med`): the final probability bound has the wrong event. The argument gives a bound by `P(\mathcal O \cup \mathcal D^c)`, not `P(\mathcal O^c \cup \mathcal D^c)`.
- `reference.tex:477-491` (`lem:small`): the McDiarmid step is valid, but only after making the independence mechanism explicit. For `v вҲү ПҖ_R(I) вҲӘ ПҖ_B(I)`, the variable `Y_v` depends only on edges between `v` and `ПҖ_R(I)` or `ПҖ_B(I)`; these edge sets are disjoint across different `v`, so the `Y_v` are independent.
- `reference.tex:557` (`lem:huge`): `N_3(v_1)` is undefined. The intended bound is `x_{v_1} вүӨ |N_{v_1}| вүӨ (1 + Оө_2)pn`, or equivalently `|\pi_B(X_{v_1}(I))| вүӨ (1 + Оө_2)pn`.
- `reference.tex:578-580` (`lem:huge`): the expression `ПҖ(I) \setminus (\bigcup X_{v_i}(I))` is ill-typed because `ПҖ(I) вҠҶ V_R Г— V_B` while `X_{v_i}(I) вҠҶ V(G)`. The intended inequality is
  `|ПҖ_R(I)| вүӨ |ПҖ_R(\bigcup X_{v_i}(I))| + |I \setminus \bigcup X_{v_i}(I)|`,
  which then yields `|\bigcup X_{v_i}(I)| вүӨ k - |ПҖ_R(I)| + o(k)`.
- `reference.tex:533-580` (`lem:huge`): beyond the two corrections above, the manuscript keeps the huge-case union-size and projected-codegree losses only at qualitative `o(k)` scale. It never names explicit blue/red slack parameters for those two witness-error pieces. The Lean development therefore has to expose exact branchwise smallness hypotheses for those pieces instead of recovering a single canonical constant from the text. With only the displayed scale assumptions, one can formalize at best the coarse bounds `paperHugeWitnessDegreeBranchParam вүӨ 3Оә`, `paperHugeWitnessCodegBranchParam вүӨ 3Оә`, and hence `paperHugeWitnessBranchParam вүӨ 6Оә`; the genuinely small branch slacks needed by the final huge-case route require an extra large-`n` specialization that the manuscript leaves implicit. Both qualitative pieces are now quantified separately in Lean via `paperHugeWitnessDegreeBranchParam_eventually_le` and `paperHugeWitnessCodegBranchParam_eventually_le`, and the combination lemmas `paperHugeWitnessBranchwisePieceBranchParamBounds_eventually_le` / `paperHugeWitnessBranchwiseBranchParamBounds_eventually_le` package them into common-threshold blue/red branch-slack bounds. The current concrete Lean specialization takes each exact piece to be at most `ОөвӮҒ Оә`, hence each branch slack to be at most `2 ОөвӮҒ Оә`; this reconnects the sharpened exact huge-case route under the stronger budget `ПҒ + (1+ОөвӮӮ)ОІ + 4 ОөвӮҒ Оә + ОҙsumGap + Оҙgap вүӨ Оә`. Lean now packages that specialization behind the canonical threshold `paperHugeWitnessTwoEpsBranchPieceThreshold ОөвӮҒ Оә ОІdeg qcodeg`, so the exact huge-case theorem can be applied with one explicit `n`-largeness hypothesis instead of a separately chosen witness `N`. More precisely, the comparison wrappers `...of_eventualTwoEpsBranchPieceBounds_of_extraDeficit` and `...of_twoEpsBranchPieceThreshold_of_coarseBudget` show that the sharpened route can replace the older coarse `6 * paperK Оә` path as soon as the later paper arithmetic provides explicit branchwise extra deficits `ОҙextraR` and `ОҙextraB` with `2 ОөвӮҒ Оә вүӨ ОҙextraR`, `2 ОөвӮҒ Оә вүӨ ОҙextraB`, `ПҒ_R + (1+ОөвӮӮ)ОІ + 2 ОөвӮҒ Оә + ОҙextraR + ОҙsumGap + Оҙgap_R вүӨ Оә`, and `ПҒ_B + (1+ОөвӮӮ)ОІ + 2 ОөвӮҒ Оә + ОҙextraB + ОҙsumGap + Оҙgap_B вүӨ Оә`. What still remains implicit in the manuscript is where those extra branch deficits are supposed to come from, or equivalently whether the displayed argument truly intends the stronger `4 ОөвӮҒ Оә` budget at this point.
- `reference.tex:639` (`lem:RISI`): `U_B` should be `T_B вҲ© E(G_B)`, not `T_B вҲ© E(G_R)`.
